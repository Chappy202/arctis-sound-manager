# Implementation Plan — Tauri v2 UI for Arctis Sound Manager

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to execute this plan task-by-task with review checkpoints. Each task is a self-contained unit of work with explicit verification. Tasks marked **[CI]** are CI/subagent-buildable and -verifiable (no webkit deps). Tasks marked **[OWNER-RUN]** require host system changes (`sudo dnf install …`) and/or the webkit GTK stack and the GUI — they are **manual and NOT auto-executed** by a subagent. A subagent may WRITE the code for OWNER-RUN tasks, but it must NOT claim the code compiles/runs; the owner compiles `src-tauri` and runs `tauri dev`/`tauri build` and verifies the GUI by hand.

---

## Goal

Ship a thin-but-real vertical slice of a Tauri v2 desktop UI for the Linux SteelSeries Arctis sound manager, scaffolded with the **official Tauri CLI** (`tauri init`) and the **official Svelte + TypeScript + Vite + pnpm** frontend stack. The app the owner can launch with `pnpm tauri dev` must:

- Show live engine state pulled from the daemon (active profile, channels, routes, device fields).
- Switch the active profile (Profiles dropdown → `switch-profile`).
- Edit per-channel parametric EQ bands live by dragging dots on an EQ curve canvas (→ `set-eq-band`), the headline feature.
- Show a Device status panel that degrades gracefully while real device data is stubbed (`device_present:false`).
- Receive live updates pushed from the backend via a Tauri event (`state-changed`).

The UI is a **client** of the existing daemon (the daemon owns all audio resources). No audio logic moves into the UI; the engine stays Tauri-free.

## Non-goals (explicitly deferred)

- Spatial / surround UI (page stubbed in nav only, no controls).
- Mic-tuning page detail (no live mic meter, no noise-gate UI).
- Streamer mode.
- Per-app routing drag-and-drop UI. A **simple route list + add-route form** is in scope only if cheap (it maps directly to the existing `route` verb); full drag-drop is future.
- OTA / auto-update UI (Plan 8).
- **Per-channel volume / mute / level meters** — the engine/daemon do **not** currently expose these (`Request` has no volume/mute verb; `ChannelSnapshot` has no volume field). Channel strips render channel identity + output device + EQ access only. Volume/mute are noted as **future** in the UI (disabled controls with a "not yet supported" tooltip, per DESIGN.md "disable and explain").
- **Per-channel output-device selection from the UI**: there is **no daemon verb** for this today (the CLI sets output directly via `ChannelManager`, bypassing the daemon protocol). This plan therefore renders the channel's current `output_device` read-only and defers a writable selector until a daemon verb exists. (See Open Questions.) A `set_route` command IS included because the `route` verb exists.
- Light theme (DESIGN.md is dark-only v1; tokens authored themeable but no light theme).
- Save / Save-as / Rename / Delete profile actions (the daemon has no verbs for these; only `switch-profile`). Dropdown shows the list + switches; management is future.

## Global Constraints

- **Official CLI tooling only** — scaffold via `@tauri-apps/cli` `tauri init` + `pnpm create vite`; no hand-rolled `src-tauri` skeleton.
- **Docs-standard structure** — `src-tauri/{Cargo.toml,build.rs,tauri.conf.json,src/main.rs,src/lib.rs,capabilities/default.json}`; `lib.rs::run()` called by `main.rs`.
- **Frontend stack is Svelte + TypeScript + Vite + pnpm** — no other framework.
- **`arctis-engine` stays Tauri-free** — no `tauri` dependency anywhere in `crates/`.
- **`arctis-client` is reused by both `asm-cli` and `src-tauri`** — the protocol types + Unix-socket client live in ONE crate; zero duplication (G1).
- **`src-tauri` build is OWNER-RUN** — it links webkit2gtk-4.1/javascriptcoregtk-4.1/libsoup-3.0 which are MISSING on the machine; CI/subagents only verify the frontend `pnpm build` and the `arctis-client` crate.
- **Typed errors that are `serde::Serialize`** — every `#[tauri::command]` returns `Result<T, CommandError>` where `CommandError` is a `thiserror` enum serialized to its string form.
- **Capabilities: add `core:event:default`** to the main-window capability so the frontend may use the event API; custom commands need NO permission.
- **48 kHz / sample-rate concerns are N/A here** — this is a UI/IPC plan; DSP lives in `arctis-audio`/daemon, untouched.

```rust
// crates/client/src/protocol.rs
pub fn socket_path() -> std::path::PathBuf;

#[derive(Debug, PartialEq, serde::Deserialize, serde::Serialize)]
#[serde(tag = "cmd", rename_all = "kebab-case")]
pub enum Request {
    GetState,
    SwitchProfile { name: String },
    SetEqBand { channel: String, band: usize, kind: String, freq_hz: f32, q: f32, gain_db: f32 },
    Route { app_binary: String, target_sink: String },
    Reload,
    Shutdown,
}

#[derive(Debug, serde::Serialize, serde::Deserialize)]
pub struct Response {
    pub ok: bool,
    #[serde(skip_serializing_if = "Option::is_none")]
    pub state: Option<arctis_engine::EngineState>,
    #[serde(skip_serializing_if = "Option::is_none")]
    pub error: Option<String>,
}

// crates/client/src/client.rs
#[derive(Debug, thiserror::Error)]
pub enum ClientError {
    #[error("connect to {path}: {source}")]
    Connect { path: String, #[source] source: std::io::Error },
    #[error("io: {0}")]
    Io(#[from] std::io::Error),
    #[error("serde: {0}")]
    Serde(#[from] serde_json::Error),
    #[error("daemon error: {0}")]
    Daemon(String),
}

/// One-shot blocking client: connect → write one request line → read one response line.
pub fn send_request(req: &Request) -> Result<Response, ClientError>;
/// Same, but to an explicit socket path (used by tests + src-tauri).
pub fn send_request_to(path: &std::path::Path, req: &Request) -> Result<Response, ClientError>;
```

```rust
    send_request_to(&socket_path(), req)

pub fn send_request_to(path: &Path, req: &Request) -> Result<Response, ClientError> {
    let stream = UnixStream::connect(path).map_err(|source| ClientError::Connect {
        path: path.display().to_string(),
        source,
    })?;
    let req_str = serde_json::to_string(req)?;
    let mut writer = stream.try_clone()?;
    writeln!(writer, "{req_str}")?;
    let mut reader = BufReader::new(stream);
    let mut line = String::new();
    reader.read_line(&mut line)?;
    Ok(serde_json::from_str(line.trim())?)
}
```

```rust
use arctis_client::{send_request_to, Request};
use std::io::{BufRead, BufReader, Write};
use std::os::unix::net::UnixListener;

#[test]
fn send_request_to_round_trips() {
    let dir = std::env::temp_dir().join(format!("asm_client_test_{}", std::process::id()));
    std::fs::create_dir_all(&dir).unwrap();
    let sock = dir.join("echo.sock");
    let _ = std::fs::remove_file(&sock);
    let listener = UnixListener::bind(&sock).unwrap();

    let server = std::thread::spawn(move || {
        let (stream, _) = listener.accept().unwrap();
        let mut writer = stream.try_clone().unwrap();
        let mut reader = BufReader::new(stream);
        let mut line = String::new();
        reader.read_line(&mut line).unwrap();
        assert!(line.contains("get-state"));
        writeln!(writer, r#"{{"ok":true}}"#).unwrap();
    });

    let resp = send_request_to(&sock, &Request::GetState).unwrap();
    assert!(resp.ok);
    server.join().unwrap();
    let _ = std::fs::remove_dir_all(&dir);
}
```

```rust
use crate::error::CommandError;
use crate::state::DaemonState;
use arctis_client::{send_request_to, Request, Response};
use arctis_engine::EngineState;
use tauri::State;
use tokio::sync::Mutex;

async fn call(state: &State<'_, Mutex<DaemonState>>, req: Request) -> Result<EngineState, CommandError> {
    let socket = { state.lock().await.socket.clone() };
    let resp: Response = tauri::async_runtime::spawn_blocking(move || send_request_to(&socket, &req))
        .await
        .map_err(|e| CommandError::DaemonUnavailable(format!("join: {e}")))??;
    if resp.ok {
        resp.state.ok_or_else(|| CommandError::Daemon("ok response missing state".into()))
    } else {
        Err(CommandError::Daemon(resp.error.unwrap_or_else(|| "unknown".into())))
    }
}

#[tauri::command]
pub async fn get_state(state: State<'_, Mutex<DaemonState>>) -> Result<EngineState, CommandError> {
    call(&state, Request::GetState).await
}

#[tauri::command]
pub async fn switch_profile(name: String, state: State<'_, Mutex<DaemonState>>) -> Result<EngineState, CommandError> {
    call(&state, Request::SwitchProfile { name }).await
}

#[tauri::command]
pub async fn set_eq_band(
    channel: String, band: usize, kind: String, freq_hz: f32, q: f32, gain_db: f32,
    state: State<'_, Mutex<DaemonState>>,
) -> Result<EngineState, CommandError> {
    call(&state, Request::SetEqBand { channel, band, kind, freq_hz, q, gain_db }).await
}

#[tauri::command]
pub async fn set_route(app_binary: String, target_sink: String, state: State<'_, Mutex<DaemonState>>) -> Result<EngineState, CommandError> {
    call(&state, Request::Route { app_binary, target_sink }).await
}
```
(Note: `send_request_to` returns `Result<Response, ClientError>`; `?` after the `spawn_blocking` join applies the `From<ClientError>` conversion via the outer `?`. The double `??` is: join error → inner `Result`.)

4. `src-tauri/src/lib.rs` — manage state, register ALL commands in one `generate_handler!`, and spawn the poll task in `.setup()` that emits `state-changed`:
```rust
mod commands;
mod error;
mod state;

use state::DaemonState;
use tauri::{Emitter, Manager};
use tokio::sync::Mutex;

#[cfg_attr(mobile, tauri::mobile_entry_point)]
pub fn run() {
    tauri::Builder::default()
        .manage(Mutex::new(DaemonState::new()))
        .invoke_handler(tauri::generate_handler![
            commands::get_state,
            commands::switch_profile,
            commands::set_eq_band,
            commands::set_route,
        ])
        .setup(|app| {
            let handle = app.handle().clone();
            tauri::async_runtime::spawn(async move {
                let mut ticker = tokio::time::interval(std::time::Duration::from_secs(2));
                loop {
                    ticker.tick().await;
                    let socket = {
                        let st = handle.state::<Mutex<DaemonState>>();
                        let guard = st.lock().await;
                        guard.socket.clone()
                    };
                    let result = tauri::async_runtime::spawn_blocking(move || {
                        arctis_client::send_request_to(&socket, &arctis_client::Request::GetState)
                    })
                    .await;
                    if let Ok(Ok(resp)) = result {
                        if resp.ok {
                            if let Some(engine_state) = resp.state {
                                let _ = handle.emit("state-changed", &engine_state);
                            }
                        }
                    }
                    // daemon-down ticks are silent; the UI keeps its last good state.
                }
            });
            Ok(())
        })
        .run(tauri::generate_context!())
        .expect("error while running tauri application");
}
```
