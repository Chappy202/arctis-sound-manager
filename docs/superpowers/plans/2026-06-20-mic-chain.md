# Microphone DSP Chain — "Clean Mic" virtual source

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` to execute this plan task-by-task with review checkpoints. Each TASK below is a self-contained TDD unit (write the failing test, make it pass, refactor). OWNER-RUN tasks (real-PipeWire/audible mic validation, optional plugin installs) are manual, NOT auto-executed — they require a human with a connected Arctis headset and a live PipeWire session, and are marked **[OWNER-RUN]**.

## Goal

Fix the "tinny / hit-or-miss mic" from the old app. We expose a virtual `Audio/Source` named **"Clean Mic"** that apps select as their microphone input. It is fed by the real hardware mic and runs an **ordered DSP chain**. The default is **CLEAN PASSTHROUGH** — no processing, no external plugin required. Every DSP stage is **opt-in** with conservative, anti-tinny defaults.

The single biggest cause of "tinny" mic audio is over-aggressive noise suppression that musical-noises the speech and pumps the spectrum. Our defense: (a) ship passthrough by default; (b) when RNNoise is enabled, default to a **conservative VAD threshold + generous grace periods** so suppression does not chew into speech; (c) offer a **parametric mic EQ** to restore body (low-shelf lift + gentle high-shelf cut) using the *existing* EQ machinery; (d) keep every stage individually toggleable so a user can bisect what made them sound bad.

This plan **generalizes** the existing filter-chain SINK generator (used by the EQ sinks in earlier plans) into one that can emit an **ordered chain of mixed node types** (PipeWire builtin nodes + LADSPA plugin nodes) and terminate in an `Audio/Source` instead of an `Audio/Sink`. We reuse `CommandRunner`/`spawn_owned`/`ChildOwner`/`AudioBackend` lifecycle and the live-Props update path. We do **not** fork the audio engine.

## Architecture

### The source filter-chain (what we build in PipeWire)

A single `libpipewire-module-filter-chain` instance, run as a dedicated `pipewire -c <conf>` child (same lifecycle model as the EQ sinks), where:

- `capture.props` binds the **real hardware mic** (`media.class = Audio/Source`, `stream.capture.sink = false`, and a `target.object` pinned to the hardware mic node name so we capture from the headset, not the default source).
- `playback.props` declares the **virtual Clean Mic** with `media.class = Audio/Source` and our stable `node.name` — this is the node apps select as their input device.
- `filter.graph` is the ordered chain of enabled stages. Stage order is fixed (signal-flow correct):

  ```
  [capture: hw mic]
     → gain (builtin "linear")            # opt-in, default off (0 dB)
     → highpass (builtin "bq_highpass")   # opt-in, default off (~90 Hz)
     → rnnoise (ladspa noise_suppressor)  # opt-in, default off, CONSERVATIVE VAD
     → compressor (ladspa sc4m)           # OPTIONAL, only if detected, default off
     → noisegate (builtin "noisegate")    # opt-in, default off, conservative thresh
     → mic EQ (builtin bq_* bands)        # opt-in, default off, reuse EqModel
  [playback: Clean Mic source]
  ```

  When **no** stage is enabled (default), the graph is a single passthrough `linear` node at unity gain — no external plugin is touched, so passthrough works even if RNNoise is missing.

- **48 kHz enforced** (`default.clock.rate = 48000`, `audio.rate = 48000`). RNNoise *requires* 48000 Hz; we always pin it.
- **Channels:** the chain is mono (`audio.channels = 1`, `audio.position = [ MONO ]`) because the headset mic is mono and RNNoise's mono variant (`noise_suppressor_mono`) is the safe path. (Stereo-aware capture is a documented future extension; the mono path is what ships.)

### The generalized generator (`crates/audio`)

Today `render_filter_chain_conf(&SinkSpec, &EqModel)` hard-codes: only `bq_*` band nodes, `media.class = Audio/Sink`, stereo `[ FL FR ]`, and a passive playback tail. We introduce a **node-list IR** that describes an ordered chain of arbitrary nodes, plus a `ChainSpec` describing the endpoints, and one renderer that both the existing EQ sink and the new mic source use.

- `FilterNode` — an enum-ish description of one graph node: its stable `name`, its `type` (`builtin` or `ladspa`), its `label`, optional `plugin` path (ladspa only), and an ordered list of `(control_name, value)` pairs.
- `ChainSpec` — endpoints + media class: `node_name`, `description`, `capture` (media class + optional `target.object`), `playback` (media class + optional `target.object` + passive flag), `channels` (`Mono`/`Stereo`).
- `render_chain_conf(&ChainSpec, &[FilterNode]) -> Result<String>` — renders the full `pipewire -c` conf. Links the nodes head-to-tail; `inputs = [ "<first>:<in-port>" ]`, `outputs = [ "<last>:<out-port>" ]`. Builtin biquad/linear/noisegate use `:In`/`:Out`; LADSPA nodes use the plugin's **actual port names** (e.g. RNNoise mono uses `Input`/`Output`), so each `FilterNode` carries its in/out port names.
- The existing `render_filter_chain_conf` is **reimplemented as a thin adapter** over `render_chain_conf` (build the `bq_*` `FilterNode` list from `EqModel`, `ChainSpec` with `Audio/Sink` + stereo). Its existing exact-fixture test (`eq_sink_3band.conf`) must still pass byte-for-byte — that is our regression guard that the generalization didn't change sink output.

### The mic source in the engine (`crates/engine`)

`reconcile()` already brings channels up, applies EQ, sets outputs, and writes routes — tracking every spawned `pipewire -c` child in `ChildOwner`. We add a step that builds/teardowns the **Clean Mic** source from the active profile's mic-chain config:

- If mic-chain is **disabled** (or config absent): ensure the source is removed (idempotent).
- If **enabled**: detect availability of each enabled stage's plugin; build the `FilterNode` list from enabled+available stages; render + spawn via `spawn_owned`; track the child token in the **same `ChildOwner`** so shutdown kills it. A stage whose plugin is missing is **dropped from the chain and reported unavailable** — the chain still builds with the remaining stages (and passthrough always builds).
- Live tuning: stages that map to in-place SPA controls (gain, highpass freq, RNNoise VAD, gate threshold, mic EQ bands) are updated via `pw-cli s <id> Props { params = [ ... ] }` without a respawn (same path as `apply_band`). Enabling/disabling a *stage* (graph topology change) requires a **recreate** (teardown + respawn) — that is acceptable and explicit.

### Data flow

```
GUI Mic page / asm-cli `mic ...`
   → daemon Request (MicGetState | MicStage | MicSet | MicEqBand)
   → Engine::mic_* methods  (mutate active profile's MicChainConfig, persist, apply)
       → convert: MicChainConfig → Vec<FilterNode> (+ availability detection)
       → AudioBackend (generalized): render_chain_conf → spawn_owned / recreate / Props
       → ChildOwner.track(token)
   → EngineState.mic snapshot (enabled stages, params, per-stage availability)
   → Response.state → GUI/CLI render (honest about unavailable stages)
```

## Research basis

- **noise-suppression-for-voice (RNNoise LADSPA, werman):** verified on-machine with `analyseplugin /usr/lib64/ladspa/librnnoise_ladspa.so`. Mono plugin label `noise_suppressor_mono`; audio ports named **`Input`** / **`Output`**; control ports **`VAD Threshold (%)`** (0–99, default 49.5), **`VAD Grace Period (ms)`** (0–1000, default 500), **`Retroactive VAD Grace (ms)`** (0–200, default 100), plus two unused `Placeholder` controls. RNNoise operates only at 48 kHz. It has **no attenuation-limit knob** (unlike DeepFilterNet) — so the only way to keep it from sounding tinny is conservative VAD + grace, which is why our default keeps it OFF and, when on, uses a permissive threshold and long grace.
- **PipeWire filter-chain builtin nodes:** `linear` (gain via `Mult`/`Add` controls), `bq_highpass`/`bq_peaking`/`bq_lowshelf`/`bq_highshelf` (`Freq`/`Q`/`Gain` controls; `:In`/`:Out` ports), `noisegate`, `dcblock` — all part of `libpipewire-module-filter-chain`, addressed live as `<node-name>:<Control>` (the same convention the EQ Props path already uses, confirmed in the EQ sink work).
- **Tinny-cause analysis:** the failure mode is aggressive single-pass NS that introduces musical noise + spectral pumping, compounded by a high-pass set too high and a gate that clips word onsets. Mitigation = passthrough default, conservative VAD/grace, opt-in stages, and a body-restoring EQ.
- **swh `sc4m` compressor:** verified on-machine; ports include `Threshold level (dB)` (-30..0), `Ratio (1:n)` (1..20), `Attack time (ms)`, `Release time (ms)`, `Makeup gain (dB)` (0..24), `RMS/peak`, `Knee radius (dB)`, audio `Input`/`Output`. Present here but treated as detected-optional.

## Non-goals (DEFER)

- **DeepFilterNet** (`libdeep_filter_ladspa`) — not installed. It is *better* than RNNoise (ML denoiser with an **attenuation-limit knob** that directly caps how much it can suppress, which is the cleanest anti-tinny control). **Recommended optional upgrade:** if/when installed, add it as an alternative suppression stage; its attenuation cap would let us relax the conservative-VAD workaround.
- **LSP plugins** (compressor/limiter/de-esser suite) — not installed. The swh `sc4m` compressor is the detected fallback; an LSP-based compressor/limiter is a future optional stage.
- **De-esser** — deferred (no installed plugin; would pair well with DeepFilterNet later).
- **Sidetone**, **mic-mute**, **mic hardware gain** — these are **device-side** (HID), handled in the device plan (Plan 2), not the DSP chain.
- **Stereo capture path** — ships mono; stereo is a documented extension.

## Global Constraints

- **Default = clean passthrough**: a fresh/disabled mic-chain config builds a single unity-gain `linear` node and needs **no external plugin**.
- **48 kHz everywhere**: `default.clock.rate = 48000`, `audio.rate = 48000`; RNNoise requires it and we always pin it.
- **Every DSP stage is opt-in** with conservative anti-tinny defaults (highpass off; RNNoise off w/ permissive VAD + long grace when on; gate off w/ conservative threshold; mic EQ off).
- **Plugin reality**: RNNoise **is** installed (`/usr/lib64/ladspa/librnnoise_ladspa.so`, `noise_suppressor_mono`); swh `sc4m` compressor **is** installed (optional/detected); **DeepFilterNet and LSP are NOT** installed.
- **Reuse, don't fork (G1)**: generalize `render_filter_chain_conf` → `render_chain_conf`; reuse `EqModel`, `CommandRunner`/`spawn_owned`, `AudioBackend`, `ChildOwner`, the live-Props path.
- **Typed errors, no panics on runtime paths**: no `unwrap`/`expect` outside tests; surface failures via `AudioError`/`EngineError`; unavailable stages are *reported*, never a crash.
- **Detect missing plugins gracefully**: builtin nodes are always available; LADSPA stages are availability-checked (file exists); a missing plugin for an enabled stage → stage dropped + marked unavailable, chain still builds.
- **Idempotent create/remove (G3)**: stable `node.name`; create is a no-op when the source exists; remove is a no-op when absent.

```rust
/// One node in a filter-chain graph. `port_in`/`port_out` are the node's actual
/// SPA port names: builtin biquad/linear/noisegate use "In"/"Out"; LADSPA nodes
/// use the plugin's real port names (e.g. RNNoise mono = "Input"/"Output").
#[derive(Debug, Clone, PartialEq)]
pub struct FilterNode {
    pub name: String,
    /// "builtin" or "ladspa".
    pub node_type: NodeType,
    /// builtin label (e.g. "bq_highpass", "linear", "noisegate") OR ladspa label.
    pub label: String,
    /// LADSPA plugin .so path; None for builtin.
    pub plugin: Option<String>,
    pub port_in: String,
    pub port_out: String,
    /// Ordered control name → value (rendered into `control = { ... }`).
    pub controls: Vec<(String, f32)>,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum NodeType {
    Builtin,
    Ladspa,
}

impl NodeType {
    fn as_str(&self) -> &'static str {
        match self {
            NodeType::Builtin => "builtin",
            NodeType::Ladspa => "ladspa",
        }
    }
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum ChainChannels {
    Mono,
    Stereo,
}

impl ChainChannels {
    fn count(&self) -> u32 {
        match self {
            ChainChannels::Mono => 1,
            ChainChannels::Stereo => 2,
        }
    }
    fn position(&self) -> &'static str {
        match self {
            ChainChannels::Mono => "MONO",
            ChainChannels::Stereo => "FL FR",
        }
    }
}

/// Endpoint media classes + targets for a generic filter-chain instance.
#[derive(Debug, Clone, PartialEq)]
pub struct ChainSpec {
    pub node_name: String,
    pub description: String,
    pub channels: ChainChannels,
    /// capture.props media.class (e.g. "Audio/Sink" for an EQ sink,
    /// "Audio/Source" for the Clean Mic — capture side binds the hw mic).
    pub capture_media_class: String,
    /// Optional capture target.object (the hw mic node.name for the mic source).
    pub capture_target: Option<String>,
    /// playback.props media.class (e.g. "Audio/Source" for the Clean Mic;
    /// for the EQ sink the playback tail is the hw sink output, class empty).
    pub playback_media_class: Option<String>,
    /// Optional playback target.object (the hw sink for an EQ sink).
    pub playback_target: Option<String>,
    /// Whether playback.props should carry node.passive = true (EQ sink tail).
    pub playback_passive: bool,
    /// The playback node.name suffix; EQ sink uses "<name>.output", the mic
    /// source uses the bare "<name>" (the source IS the playback side).
    pub playback_node_name: String,
}
```
